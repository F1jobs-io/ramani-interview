Clinical Trial Data Model (Admin-Oriented)

  • Study, Protocol, Protocol Version, Site, Investigator (PI), Subject/Patient, Screening, Enrollment, Visit, Adverse Event, Consent.
  • Snapshot Protocol Version on Enrollment; maintain version lineage across visits.
  • Prevent multiple active enrollments per subject; dedupe with hashed identifiers and supervised merges.
  • Security: OWD Private for Subject/AE; role hierarchy by Program→Study→Site; Restriction Rules for PHI.
